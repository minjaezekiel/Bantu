# Bantu v1.3.0 — Status Report

A core-language correctness release: features that were advertised via keywords but silently
failed now work, the parser can no longer spin on a bad token, and several requested capabilities
land. **Object-oriented features were intentionally left untouched.**

Verified by rebuilding the interpreter (`bantu-src/compiler/build-mac.sh`) and running the test
suites below on macOS. `bantu --version` → `Bantu v1.3.0`.

---

## ✅ What works (with test evidence)

All items below are covered by `tests/lang_test.b` (**23/23 passing, "ALL GREEN"**) unless noted.

### Fixed semantics
| Feature | Before | Now |
|---|---|---|
| Compound assignment `+= -= *= /=` | `[RUNTIME ERROR] Unknown operator`; infinite loop in `for` | Applies the base operator; loops terminate |
| `break` / `continue` | Inert no-ops | Exit / skip loop iterations correctly |
| `switch` / `case` / `default` | Unimplemented (`Unexpected token 'switch'`) | Works; braces required, no fallthrough |
| `throw` + `try` / `catch` | Never caught runtime errors | Catches runtime errors *and* thrown values (`$e` = `{message,type,line}`) |
| Parser error recovery | Could loop forever on one syntax error | Reports once, synchronizes, compile gate blocks run |
| `const` | Reassignable (bug) | Truly final — reassignment is a compile *and* runtime error |

### New capabilities
| Feature | Example |
|---|---|
| Python-style dict iteration | `for $k, $v in $dict.items() { }` |
| Dict methods / builtins | `$d.keys()` `$d.values()` `$d.items()` `$d.size()`; `keys/values/entries` |
| List mutators (in place) | `append` `pop` `insert` `remove` `extend` — and `push()` / `$l.push()` / `$l.pop()` now mutate for real (they were silent no-ops) |
| Python-style file I/O | `open/read/readline/readlines/write/close` + `readfile/writefile/appendfile` |
| FFI via libffi | `func(loadlib("libm.dylib"), "sqrt", "double", ["double"])(16.0)` → `4` |
| Parameterized SQL | `sua.sqlite.query("… WHERE age > ?", [18])` — injection-safe. PostgreSQL binds via `PQexecParams` (`$1…$n`). |
| **ORM binds every value** | `orm/orm.b` no longer inlines escaped literals — WHERE/INSERT/UPDATE values are bound parameters. Injection payloads round-trip as data (tested). |
| Linter + compile gate | `bantu lint file.b --json`; `run`/`build` refuse on errors (`--no-lint` opts out) |

### Bonus fixes (old lenient parser had masked these)
- `$db`, `$create`, `$list`, `$switch`, … — reserved words usable as variable names (the `$`
  sigil forces an identifier).
- Anonymous functions — `def($a, $b) { … }` as a first-class value.
- Bare `return;` — returns null instead of a parse error.
- `.size()` on lists/dicts/strings.

### Regression suites (unchanged, still green)
- **ORM:** `bantu run orm/orm_test.b` → **46/46 PASS**.
- **ORM demo:** `bantu run samples/orm-demo/main.b` → runs clean.
- **OOP:** classes/inheritance/`this`/`super` untouched and working.

## ⚠️ Known issues / not done

| Item | Status | Notes |
|---|---|---|
| **`sua.mysql.*` is a SIMULATION** | **Not a real driver** | Newly discovered: `HAS_MYSQL` is defined by CMake under `-DBANTU_MYSQL=ON`, but the evaluator **never references it** — `sua.mysql.query` always returns canned rows (Alice/Bob) and never contacts a server. `drivers/mysql_driver.hpp` and `drivers/postgres_driver.hpp` are **included by nothing** (dead code). The README previously claimed MySQL was "real with `-DBANTU_MYSQL=ON`"; that claim is now corrected. PostgreSQL *is* real (inline `HAS_LIBPQ` + libpq). Wiring a real MySQL driver is open work. |
| `samples/blogsite` | **Fails at startup** | Pre-existing **cross-module closure bug** — route handlers defined in `routes.b` reference `postController` from `server.b`'s scope. The stricter (fatal) error model *exposed* this; it is not a regression in the language features. Namespaced function re-export itself was verified working. Fixing the sample's architecture is out of scope for this release. |
| Windows FFI | **Stubbed** | `loadlib`/`func` raise a clear "FFI not available" error on Windows; Linux/macOS ship it enabled (`-DBANTU_FFI -lffi -ldl`). |
| Release CI (`release.yml`) | **First run pending** | Never executed before v1.3.0 (no tags existed). The Windows job (vcpkg + WiX MSI) is the most likely to need a fix on its first attempt. |
| Type annotations (`number $x = …`) | **Not runtime-enforced** | Still advisory at runtime; the linter emits a **warning** on obvious literal mismatches. Only `const` is hard-enforced. |
| Bytecode VM / performance | **Deferred** | Interpreter remains tree-walking; a VM would rewrite the OOP dispatch and was deferred by decision. |
| `async` / `await` | **Deferred** | Reserved keyword, still unimplemented. |
| Native binary compilation | **Deferred** | `bantu build` still emits a bash wrapper. |

## How to verify

```sh
# rebuild
bash bantu-src/compiler/build-mac.sh          # macOS  (or build.sh on Linux)

# language features
bantu run tests/lang_test.b                   # → PASS: 23  FAIL: 0  ALL GREEN

# ORM regression
bantu run orm/orm_test.b                      # → PASS: 46  FAIL: 0  ALL GREEN

# linter + compile gate
bantu lint tests/const_bad.b                  # → 1 error, 1 warning
bantu run  tests/const_bad.b ; echo $?        # → blocked, exit 1
```

See `CHANGELOG.md` (`[1.3.0]`, tagged `[feature]`/`[bug fix]`/`[patch]`) and
`docs/language-features.md` for full details.
